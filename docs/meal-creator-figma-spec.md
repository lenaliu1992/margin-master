# 编辑套餐弹窗 Figma 高保真设计规范

适用对象：
- 产品经理：确认信息层级与交互目标
- 设计师：直接在 Figma 中搭建高保真稿
- 前端工程师：据此实现 [MealCreator.tsx](/Users/wenshuaibi/Documents/claude/tuangou/tuangou-nextjs/components/MealCreator.tsx)

设计目标：
- 做成专业 SaaS 后台式弹窗，而不是“好看但难用”的视觉拼贴
- 第一眼能读懂：左边选菜，右边定价和毛利
- 让“编辑套餐”成为高频操作界面，耐看、稳定、可快速扫描

## 1. Frame 与栅格

### Desktop 主稿
- Frame 名称：`编辑套餐弹窗 / Desktop`
- 尺寸：`1440 x 960`
- 弹窗外边距：上下 `24`，左右 `24`
- 主容器建议尺寸：`1368 x 912`
- Grid：`12 columns`
- Margin：左右 `48`
- Gutter：`24`

### Narrow 稿
- Frame 名称：`编辑套餐弹窗 / Narrow`
- 尺寸：`960 x 900`
- 弹窗外边距：上下 `20`，左右 `20`
- 主容器建议尺寸：`920 x 860`
- Grid：`8 columns`
- Margin：左右 `24`
- Gutter：`16`

## 2. 整体结构

弹窗结构固定为 3 段：

1. 顶部 Header
- 标题
- 一句说明
- 关闭按钮

2. 中部编辑区
- 左侧：选菜区
- 右侧：套餐信息与定价结果区

3. 底部 Footer
- 左侧状态摘要
- 右侧操作按钮

桌面端布局：
- 左栏占比约 `56%`
- 右栏占比约 `44%`
- 左右之间使用 1px 分隔线或浅阴影分区，不用重边框

窄屏布局：
- 改为上下结构
- 顺序：Header -> 套餐名称 -> 价格结果 -> 选菜区 -> Footer
- Footer 保持固定或视觉上独立，不能埋到长滚动内容里

## 3. 视觉方向

风格固定为：`高级后台`

视觉原则：
- 以白、浅灰、石墨蓝为基础
- 用少量青蓝作为主强调色
- 颜色只服务于状态与重点，不做大面积彩色背景
- 去掉中英混排
- 去掉厚重描边、过度发光、装饰性渐变和夸张阴影
- 强调“秩序感”和“信息可读性”

## 4. Design Tokens

### 颜色

主背景：
- `bg/base`: `#F5F7FA`
- `bg/modal`: `#FFFFFF`
- `bg/subtle`: `#F8FAFC`
- `bg/section`: `#F3F6F9`

文本：
- `text/primary`: `#0F172A`
- `text/secondary`: `#475569`
- `text/tertiary`: `#94A3B8`

边框：
- `border/default`: `#E2E8F0`
- `border/strong`: `#CBD5E1`
- `border/focus`: `#38BDF8`

强调色：
- `accent/primary`: `#0F4C81`
- `accent/soft`: `#E0F2FE`
- `accent/ui`: `#0891B2`

状态色：
- `success`: `#059669`
- `warning`: `#D97706`
- `danger`: `#DC2626`

### 圆角
- `radius/sm`: `16`
- `radius/md`: `20`
- `radius/lg`: `24`
- 不允许同页混入大量 `8 / 12 / 32+` 的随机圆角

### 阴影
- `shadow/card`: `0 6 20 rgba(15, 23, 42, 0.06)`
- `shadow/modal`: `0 24 80 rgba(15, 23, 42, 0.20)`

### 字体层级
- 弹窗标题：`32 / 700`
- 模块标题：`20 / 600`
- 字段标签：`13 / 600`
- 正文：`14 / 400`
- 辅助说明：`12 / 400`
- 指标数字：`32 / 700`
- 次级指标：`20 / 700`

### 间距系统
- 以 `4` 为基准
- 常用 spacing：`8 / 12 / 16 / 20 / 24 / 32`
- 大区块之间统一使用 `24`
- 卡片内统一使用 `16` 或 `20`

## 5. 顶部 Header 规范

高度建议：
- Desktop：`112`
- Narrow：`96`

组成：
- 左上角可选小标签：`编辑套餐` 或 `新建套餐`
- 主标题：`编辑团购套餐`
- 副标题：一句说明，例如“左侧选菜，右侧统一完成定价与毛利判断”
- 右上角关闭按钮

样式要求：
- Header 背景纯白
- 只允许一层很轻的背景氛围，不用彩色大片渐变
- 关闭按钮为圆角方形，不用透明悬浮 icon

## 6. 左侧选菜区规范

模块名称：
- `选菜区`

结构：
- 区块标题
- 搜索框
- 菜品滚动列表

搜索框：
- 固定在列表顶部
- 输入高度 `48`
- 左侧搜索图标，右侧不放多余按钮

菜品行信息密度：
- 菜名
- 分类
- 单点价
- 成本
- 选中状态

默认状态：
- 白底
- 1px 边框
- hover 仅轻微提亮边框和背景

选中状态：
- 不用发光、粗蓝边、厚外阴影
- 改为：
  - 轻色背景
  - 细边框强调
  - 左侧状态点或选中标记

数量控制：
- 仅在选中时展开
- 放在菜品行下方的次级区域
- 包含：
  - 减号按钮
  - 当前数量
  - 加号按钮
- 控件必须横向对齐，尺寸统一

空状态：
- 虚线框或浅底提示
- 文案：`没找到匹配菜品，换个关键词试试`

## 7. 右侧信息区规范

右侧拆成 4 个模块，按顺序排布：

### 7.1 套餐名称
- 单独卡片
- 标题：`套餐名称`
- 一个主输入框
- 输入框高度 `52`
- placeholder 示例：`例如：花生煲鸡爪 2-3 人餐`

### 7.2 已选菜品摘要
- 标题：`已选菜品`
- 内容展示为标签流，不是大段表格
- 每个标签包含：
  - 菜名
  - 数量 badge，如 `x2`
- 标签要足够紧凑，不能显得凌乱

### 7.3 定价输入
- 由两张风格统一的输入卡组成：
  - `秒杀价`
  - `官方补贴金额`
- 两张卡使用同一视觉语言：
  - 白底
  - 轻边框
  - 左侧标签与说明
  - 中间大输入框
  - 右侧结果摘要

禁止：
- 两张卡颜色风格完全不同
- 一张像表单、一张像海报
- 结果数字与输入框抢同层级

### 7.4 利润结果区
- 单独做成“决策摘要卡”
- 展示：
  - 组合总原价
  - 组合总成本
  - 秒杀价毛利率
  - 补贴后到手价
  - 最终毛利率
- 原价/成本为基础数据
- 毛利和到手价为结果数据
- 结果数据必须更醒目，但仅用一处主视觉强调

## 8. 结果表达规则

毛利率颜色规则：
- `>= 60%`：成功色
- `40% - 59.9%`：警告色
- `< 40%`：风险色

表达方式：
- 用数字 + 小 badge
- 不用整块大红、大绿背景
- 不用进度条做主视觉

到手价卡：
- 当填写补贴后才出现
- 标题：`最终到手价`
- 辅助说明：`秒杀价减去补贴后实际入账`
- 大数字只突出一项，不要多个大数字同时竞争

## 9. Footer 规范

高度建议：
- `72 - 88`

布局：
- 左侧：状态摘要
- 右侧：操作按钮

左侧文案：
- 已选几种菜、共几份
- 若未选菜，则提示先从左侧选择菜品

右侧按钮：
- 次按钮：`取消`
- 主按钮：
  - 编辑态：`更新套餐`
  - 新建态：`保存套餐`

按钮规范：
- 高度统一 `48`
- 主按钮用深色实底
- 次按钮用白底 + 边框
- 禁止使用过亮荧光色作为主按钮

## 10. 状态规范

### 输入框
- Default：白底、浅边框
- Hover：边框轻微加深
- Focus：青蓝色 focus ring
- Disabled：灰底、低对比文本

### 菜品行
- Default
- Hover
- Selected
- Selected + quantity editing

### 按钮
- Default
- Hover
- Active
- Disabled

### 毛利结果
- Success
- Warning
- Danger

## 13. 验收标准

- 第一眼就能分清：左边选菜，右边定价与毛利
- 套餐名称、价格输入、结果输出不再挤在一起
- 已选菜品摘要整洁，不像一堆无序标签
- 毛利率只是决策提示，不是页面主视觉污染源
- 桌面端可长时间编辑，不累
- 设计稿足够具体，另一个设计师或工程师无需再自行做视觉判断
